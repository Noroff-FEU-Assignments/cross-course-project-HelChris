Web Security

How would you advise a potential client to secure his/her website from web security attacks?

Some advise to a potential client to secure their website from web security attacks. To hinder XSS, XSRF, SQL Injections and other mischief with:

  • Research. Go for the well-known and well-used third-party applications/libraries.

  • HTTPS: Use secure HTTP (HyperText Transfer Protocol) by combining HTTP with TLS (Transport Layer Security) = HTTPS = secure and encrypted communication between the server and the client. So that sensitive information (login info, personal data etc.) stays protected.

  • Strong! passwords: Always use strong passwords, consider using a (well-known and well-used) password manager to help you create and safekeeping strong passwords.

  • Sanitize and validate data: after doing research implement the use of a solution like the sanitize-html package in NodeJS or an ORM(Object Related Mapping) library for SQL related code.

  • Go for all the updates! Keep the software and all the things up to date to ensure security flaws are being plugged and patched to stay ahead of hackers.

  • Secure authorization and authentication: with the use of an open-standard authorization protocol or framework like onelogin.

  • Implement Automatic user timeout: don't let the sensitive information swirl around in the browser for a long time.

  • A Content Security Policy (CSP): Restrict what content is allowed to load on the webpages.

  • Backup data on a regular basis. “In case of fire”, please keep all the website data backed up.

  • Monitor: Utilize a Web Application Firewall (WAF) to monitor and filter the incoming web traffic.

  • Educate: help the client and future users stay UpToDate on good practices regarding web security
